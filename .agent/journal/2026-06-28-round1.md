# Neon Backgammon VR — Build Journal

## Round 1 (2026-06-28, 07:00–07:17 UTC)

### Scaffold
- Copied `neon-pairs` as template (`@iwsdk/create` was hanging)
- Cleaned template code, kept vite config with `compileUIKit()` + `iwsdkDev()`

### Architecture
- Single `src/index.ts` (1,927 lines) + 15 `.uikitml` files (195 lines) = 2,122 LOC
- Full backgammon engine: legal move generation with bar re-entry, bearing off, gammon/backgammon detection, hit mechanics
- AI opponent with board evaluation heuristic (pip count, anchors, blots, primes, home board)
- 3 difficulty levels: Easy (random), Medium (eval + noise), Hard (pure eval)

### Game Modes (8)
Single, Match-3, Match-5, Daily Challenge, Blitz (30s turns), Practice, Nackgammon (initial setup variant), Hypergammon (3 checkers each)

### 3D Scene
- Board with 24 triangles (alternating colors), raised borders, borne-off trays
- 32 pooled checker meshes (16 per player), proper stacking
- 2 dice meshes, highlight rings for valid moves
- Holodeck: grid floor/ceiling, 14 floating decorations, 40 ambient particles, fog, accent lights

### PanelUI (15 panels)
title, modeselect, difficulty, hud, pause, gameover, achievements, settings, help, stats, skins, countdown, toast, diceroll, leaderboard

### Progression
- 30 achievements across gameplay milestones
- 8 piece skins, 5 color themes
- XP/level system (50 levels, scaling XP curve)
- Seeded daily challenge (mulberry32 PRNG)

### Input
- Mouse: raycasting for point selection, click to move
- XR: trigger to roll dice, B to pause, pointer for menu interaction
- Keyboard: Enter to roll, Escape to pause

### Audio
Procedural: dice roll, piece move, hit, bear off, win, lose, countdown tick, achievement unlock, ambient drone

### Tech Issues
- `HTMLElement` → `HTMLDivElement` cast needed for canvas parent
- `world.input.keyboard` and `world.input.xr` needed `as any` casts
- `canvasPointerEvents` not in WorldOptions type — used `as any` on World.create options
- Camera position set manually after World.create

### Deploy

## Round 4 (2026-06-28, 07:28–07:31 UTC)

### Features Added
- Doubling cube gameplay:
  - DOUBLE button on diceroll panel (visible when player can double before rolling)
  - `canDouble()`: checks cube ownership and value cap (64)
  - `handlePlayerDouble()`: offers double to AI, AI evaluates position to accept/decline
  - `handleAIDouble()`: AI offers double when position is strong (>150 eval advantage, 30% chance)
  - `aiAcceptsDouble()`: accepts if eval difference > -200
  - Player auto-accepts/declines based on position evaluation
  - Declined double = instant win/loss for current cube stakes
- Full-turn AI look-ahead (Hard):
  - `evaluateFullTurn()`: recursive function evaluates all possible move sequences (depth ≤ 3)
  - AI picks first move that leads to best overall turn evaluation
  - Medium still uses single-move evaluation with noise
- AI doubling decisions in `endTurn()`: 30% chance when position > 150 eval advantage
